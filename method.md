# Method: Confidence-Gated Evidence Composer (CGEC)

## Purpose

CGEC targets the highest-ROI failure mode: `signal_overclaiming_under_source_uncertainty`.

The mechanism links source reliability and signal confidence directly to message phrasing and send eligibility, reducing factual over-claims while preserving conversion performance.

## Core Mechanism

1. **Signal collection layer** builds `hiring_signal_brief.json` and `competitor_gap_brief.json` with `data_sources_checked` and per-signal confidence.
2. **Confidence-gated phrasing policy** maps confidence level to language mode:
   - high: evidence-forward statement
   - medium: probabilistic framing + verify question
   - low: question-only framing (no hard claim)
3. **Commitment guardrail** blocks outreach if any of:
   - bench mismatch detected
   - unsupported competitor-gap confidence
   - contradictory high-certainty language with weak source status
4. **Channel policy gate** enforces email-first and allows SMS only after prior email engagement.
